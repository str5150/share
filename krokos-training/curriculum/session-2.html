SESSION 02

PCの仕組みを理解する

60分 ｜ 座学 + ハンズオン
今日のゴール

ターミナルが何をする場所か、Claude Codeが自分のPC上で何をしているかを理解する

2-1

ターミナルとは

15分

普段はマウスでフォルダを開いたりファイルを削除したりしている。ターミナルは同じことを「文字」でやる場所。

【普段のやり方】                【ターミナルで同じこと】
フォルダをダブルクリック   →  cd my-project
ファイル一覧を見る         →  ls
新しいフォルダを作る       →  mkdir new-folder
ファイルを削除             →  rm file.txt

やっていることは同じ。マウスの代わりに文字で命令するだけ。

ハンズオン: ターミナルを開いて一緒にやってみる

1. ターミナルを開く（Macなら「ターミナル」アプリ）
2. pwd と打つ → 今いる場所が表示される
3. ls と打つ  → ファイル一覧が表示される
4. mkdir test-folder と打つ → フォルダが作られる
5. ls でもう一度確認 → test-folder が増えている

Finderで確認すると、ちゃんとフォルダができている。ターミナルはただの「文字版Finder」。

ターミナルでできること（意外と多い）

ファイルのダウンロード（curl）
ソフトのインストール（brew install）
サーバーの起動（npx serve）
GitHubへのアップロード（git push）

AIエージェントがターミナルを使うということは、AIがこれら全部できるということ。だから「何をしていいか」を管理する必要がある。

2-2

「実行する（Run）」と「止める」

5分 
実行 = プログラムを動かすこと
  node app.js  ← これで app.js が動く

止める = Ctrl + C
  サーバーを起動したとき、画面が止まって見えることがある。
  壊れたのではなく、動き続けているだけ。Ctrl + C で止められる。

「実行したけど画面が固まった」はよくあること。フリーズではなく、プログラムが動き続けている状態。

2-3

Homebrew・PATH・bin

10分

深入りは不要。「こういう仕組みがある」と知っておくだけでいい。

Homebrew = ターミナル版のApp Store
  → brew install node でツールをインストールできる

bin = ツール置き場
  → インストールしたツールは /usr/local/bin/ などに置かれる

PATH = ツールの探し方リスト
  → ターミナルで claude と打つと、PATHに書かれた場所を順番に探す
  → 見つからないと「command not found」エラーになる
覚えておくこと

「command not found」が出たらPCが壊れたわけではない。ツールが置いてある場所をPCが知らないだけ。トラブル時に「PATHの問題かも」と思えるだけで対応が変わる。

2-4

Claude Codeはどこで何をしているのか

15分

第2回の核心。

3つのポイント

  1. AIが触るファイルは自分のPCの中にある
  2. AIの頭脳はAnthropicのサーバーにある
  3. AIに見せた情報はインターネット経由で送信される

作業フォルダ

重要

Claude Codeは起動したフォルダの中で動く。「AIに触らせていいファイルだけが入ったフォルダ」で起動する。これが一番基本のセキュリティ。

2-5

APIとは

5分 
チャットAI = レストランで食事
  → ブラウザという店に行って、その場で質問して回答をもらう

Claude Code = 出前
  → 自分のPC（キッチン）からAPI（電話）で回答（料理）を届けてもらう
  → 届いた結果を自分のPCで好きに使える

APIキーは出前の会員番号。漏れたら他人に使われて課金が発生する。パスワードと同じ扱い。

2-6

振り返り

10分
Q1: ターミナルの ls コマンドは何をする？
ファイルの一覧を表示する。FinderやExplorerで中身を見るのと同じ。
Q2: Claude Codeを起動するとき「作業フォルダ」が重要な理由は？
AIが触れるファイルの範囲が決まるから。顧客データが入ったフォルダで起動すると危険。
Q3: 「command not found」エラーの意味は？
PATHに登録された場所にプログラムがない。PCが壊れたわけではない。

宿題

ターミナルでコマンドを打ってみる

pwd           ← 今いる場所を表示
ls            ← ファイル一覧
cd Desktop    ← デスクトップに移動
mkdir ai-work ← 「ai-work」フォルダを作る
cd ai-work    ← そのフォルダに入る
pwd           ← 場所が変わったことを確認

「AIに触らせても安全なフォルダ」と「絶対ダメなフォルダ」を書き出す

自分のPCのフォルダ構成を見て、それぞれ1つずつ書き出す。これが「作業フォルダを分ける」の実践になる。
