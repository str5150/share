SESSION 03

ブランド業務に適用する

60分 ｜ 座学 + 実践ワーク
今日のゴール

ルールファイルとMCPを使って、自社ブランドのトンマナや業務データをAIに活かす方法を身につける

3-1

AIエージェントの内部動作

10分

「商品LPを作って」と指示したら、裏でこういうことが起きている。

STEP 2〜5がループしている。サーバーで考えて、PCで実行して、結果を見て、また考えて…これが数十回〜数百回繰り返される。チャットAIは1往復。エージェントはこのループを自動で回す。

「許可」のタイミング

勝手にやること   → ファイルの読み書き
確認してくること → コマンド実行、ファイル削除
重要

確認画面を読まずにOK連打すると、セキュリティラインがなくなる。内容を確認してからOKする癖をつける。

3-2

ルールファイル ― AIに「うちのやり方」を教える

15分

チャットAIでは毎回「フォントはこれ」「色はこれ」「日本語で書いて」と指示し直す必要がある。AIエージェントにはこれを一発で解決する仕組みがある。

実態を見る

my-project/
├── index.html
├── style.css
└── ルールファイル    ← ただのテキストファイル

ルールファイルはツールによって名前が違うが、中身はどれも同じ。

ツールルールファイルの名前
Claude CodeCLAUDE.md
CodexAGENTS.md
Cursor.cursor/rules 内のファイル

中身はこういうもの。

# ブランドルール
- ブランド名は「〇〇」表記で統一
- メインカラーは#1a73e8、サブカラーは#f5f5f5
- フォントはNoto Sans JPを使う
- トーンは「親しみやすく、でも安っぽくない」
- 価格は税込表記
- 「お客様」ではなく「あなた」で呼びかける

魔法ではない。テキストファイルにルールを書いて、AIに毎回読ませているだけ。新人にマニュアルを渡すのと同じ仕組み。

「学習」ではなく「マニュアル」

チャットAIAIエージェントのルール
仕組み会話の中で覚えているテキストファイルに書いてある
次の会話忘れているファイルがある限り有効
チームで共有できないファイルを渡すだけ
中身の確認難しいファイルを開けば全部見える

AIが賢くなったのではなく、毎回マニュアルを読んでから仕事している。だから中身は誰でも確認できるし、書き換えられる。

3-3

実践①: メルマガのトンマナをルール化する

15分

ここから手を動かす。自社で過去に配信したメルマガを使って、ブランドのトンマナをAIに覚えさせる。

1. 作業フォルダを作る
   mkdir newsletter-rules && cd newsletter-rules

2. 過去のメルマガを2〜3本、テキストファイルとして保存する
   → sample-1.txt, sample-2.txt, sample-3.txt

3. AIエージェントに指示する:
   「このフォルダにあるメルマガのサンプルを読んで、
    文体・トンマナ・よく使う表現・NGワードを分析して、
    ルールファイルにブランドガイドラインとしてまとめて」

4. AIが生成したルールファイルを確認する
   → 「この表現は違うな」「この部分は合ってる」を自分で判断

5. ルールファイルを修正・追記する
   → 自分で直してもいいし、AIに「〇〇を追加して」と指示してもいい

6. ルールが入った状態で、新しいメルマガの下書きを依頼する
   → 「来月の新商品キャンペーンのメルマガ本文を書いて」
   → ルールなしの出力と比較してみる

確認ポイント

  • ルールなしの出力 → 汎用的でどのブランドにも当てはまる文面
  • ルールありの出力 → 自社のトンマナに沿った文面

この差が「ルールファイルの効果」。第1回で作ったページのスクショと今回の出力を比べると、精度の違いが実感できる。

3-4

MCP ― AIの道具箱を1つずつ組み立てる

10分 
素のAIエージェント:
  ファイルを読む / ファイルを書く / コマンドを実行する
  → これだけ。外部サービスとの連携はゼロ。

今の状態だと、AIはPCの中のファイルしか触れない。外部サービスと繋げるのがMCP。

Google Sheets MCPを追加 → スプレッドシートを読み書きできるようになる
Web Search MCPを追加   → Web検索ができるようになる
Slack MCPを追加        → Slackに投稿できるようになる

App Storeで1つずつアプリを入れるのと同じ。全部最初から入っているわけではない。

道具ごとに「できること」が決まっている

Google Sheets MCP:
  ✅ スプレッドシートの読み書き
  ❌ Gmailの送信（これは別のMCP）
  ❌ Googleドキュメント（これも別）

MCPを1個入れたら全機能が使えるわけではない。1つずつ追加する。だからリスクも1つずつコントロールできる。

3-5

実践②: DMチラシの素材を作ってみる

10分

学んだルールファイルを活用して、もう1つ実務寄りのワークに取り組む。

1. 先ほどのフォルダにDM用の素材情報を追加する
   → product-info.txt に商品名、特徴、価格、ターゲット層を書く

2. AIエージェントに指示する:
   「product-info.txt の情報を使って、
    DMチラシに使うキャッチコピー3案と、
    本文テキスト（200文字以内）を作って。
    ルールファイルのブランドガイドラインに従って。」

3. 出力を確認する
   → トンマナがブランドに合っているか
   → 商品情報が正確に反映されているか

4. 修正指示を出す
   → 「2案目のコピーをもう少しカジュアルにして」
   → 「価格の表記を税込に変えて」

ルールファイルがあるから、毎回ゼロから説明しなくていい。修正指示も「もう少しカジュアルに」程度で伝わる。

3-6

振り返り

Q1: ルールファイルの正体は？
テキストファイル。AIに毎回読ませる「ブランドマニュアル」。ツールによって名前が違う（CLAUDE.md / AGENTS.md など）。
Q2: MCPを追加するとどうなる？
AIの道具箱に新しい道具が1つ増える。できることが広がるが、アクセス範囲も広がる。
