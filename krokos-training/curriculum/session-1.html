SESSION 01

AIエージェントを知る

60分 ｜ 座学 + 実践
今日のゴール

チャットAIとAIエージェントの違いを体感し、実際に1つ指示を出してみる

1-1

生成AIの整理

10分

AIの種類が多すぎて混乱しがちだが、違いはシンプル。

AI開発元ひとことで言うと
GPTOpenAI万能型。何でもそこそこやる
ClaudeAnthropic慎重で丁寧。長文に強い
GeminiGoogle検索との連携が強い
LlamaMetaオープンソース。自社サーバーで動かせる

覚えることは1つだけ。「どのAIを使うか」より「どう使うか」が重要。

チャットAIには「エージェント版」がある

普段使っているチャットAIの多くに、ファイル操作やコマンド実行ができる「エージェント版」が存在する。

チャットAIエージェント版何が変わる？
ChatGPT（OpenAI）→Codexテキスト回答だけでなく、コードを書いて実行する
Claude（Anthropic）→Claude Codeファイルの作成・編集・削除、コマンド実行を自動でやる
Gemini（Google）→Gemini Code AssistGoogle Cloudと連携して開発作業を自動化する

チャットAIは「相談相手」。エージェントは「作業してくれる人」。同じ頭脳を持っているが、手足がついた。どのエージェントでも基本の考え方は同じなので、この講座で学ぶことはどのツールにも応用できる。

1-2

チャットAI vs AIエージェント

15分

RPAとの比較

RPAAIエージェント
たとえるとレールの上の電車自分で道を決めるタクシー
想定外のこと止まる自分で考える（たまに間違う）
指示の出し方フロー図を組む日本語で伝える
ポイント

RPAは安心だけど融通が利かない。AIエージェントは融通が利くけど、どこまで走っていいか決めてあげないと暴走する可能性がある。「範囲を決める」が今後の講座で学ぶ核心。

1-3

AIエージェントツールの種類

5分
ツールひとこと
Claude Code / Codexターミナルで動く。速い。腕利きの職人
Cursor画面付きエディタ。変更が目に見える。隣に座る助手
Devin / Antigravityクラウド上で完結。リモートの外注チーム

見た目は違うが、裏でやっていることは全部同じ（ファイルの読み書き + コマンド実行）。この講座で学ぶ考え方は、どのツールにも使える。

1-4

実践: 最初の指示を出してみる

20分

ここからは実際にAIエージェントを触る。自社ブランドの簡単な商品紹介ページを作ってみる。

手順:
  1. デスクトップに「test-page」フォルダを作る
  2. AIエージェントツール（Codex / Claude Code / Cursor など）を
     そのフォルダで起動する
  3. 以下のように指示を出す

指示の例:
  「シンプルな商品紹介ページを1枚作って。
   商品名は〇〇（自社の商品名）。
   特徴を3つ並べて、購入ボタンを置いて。」

観察ポイント

  • AIがファイルを勝手に作成している
  • エラーがあれば自分で直している
  • 「許可していいですか？」と聞いてくるタイミングがある

チャットAIなら「こういうHTMLを書いてください」とテキストが返ってくるだけ。エージェントはファイルそのものを作る。この違いを体で覚える。

1-5

振り返り

10分
Q1: チャットAIとAIエージェントの一番大きな違いは？
AIが自分でファイルを作り、コマンドを実行する。「口だけ」ではなく「手を動かす」。
Q2: RPAとAIエージェント、想定外の対応に強いのは？
AIエージェント。ただし想定外の行動をする可能性があるのでリスク管理が必要。
Q3: Codex、Claude Code、Cursor、Devinで共通していることは？
裏ではどれも「ファイルの読み書き + コマンドの実行」をしている。

宿題

今日作ったページをスクショで保存する

第3回で比較に使う。ルールなしで作った結果がどう変わるかを確認するため。

「毎回やっている定型作業」を3つ書き出す

メルマガ作成、商品説明文の更新、DM制作、SNS投稿文…など。第3回以降でAI化する対象を選ぶ材料になる。
