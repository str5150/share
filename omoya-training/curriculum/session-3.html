3-1

AIエージェントの内部動作

15分

「LPを作って」と指示したら、裏で何が起きるのかを6ステップで説明する。

核心

STEP 2〜5がぐるぐるループしている。サーバーで考えて、PCで実行して、結果を見て、また考えて…これが数十回〜数百回繰り返される。チャットAIは1往復。エージェントはこのループを自動で回す。

許可を求めてくるタイミング

勝手にやること → ファイルの読み書き
確認してくること → コマンド実行、ファイル削除
重要

この確認画面を読まずにOK連打すると、セキュリティラインが無くなる。

3-2

ルールとメモリ ― AIに「やり方」を教える

20分

チャットAIで毎回同じことを説明するの、面倒じゃないですか？ AIエージェントにはこれを解決する仕組みがある。

実態を見せる

my-project/
├── index.html
├── style.css
└── CLAUDE.md    ← これ！ ただのテキストファイル

CLAUDE.mdを実際に開いて中身を見せる。

# プロジェクトルール
- フォントはNoto Sans JPを使う
- メインカラーは#1a73e8
- ボタンは角丸8px
デモ（5分）

1. ルールなしで「ボタンを作って」→ 適当なスタイル / 2. CLAUDE.mdにルール追加 / 3. もう一度「ボタンを作って」→ ルール通りのスタイル

「学習」ではなく「マニュアル」

チャットAI AIエージェントのルール
仕組み 会話の中で覚えている テキストファイルに書いてある
次の会話 忘れている ファイルがある限り有効
チームで共有 できない ファイルを渡すだけ
中身の確認 難しい ファイルを開けば全部見える

ルールの範囲（スコープ）

グローバルルール（PC全体に適用）
  └─ プロジェクトルール（このフォルダだけ）
       └─ 会話中の指示（今の会話だけ）
3-3

MCP ― AIの道具箱を1つずつ組み立てる

15分

素のClaude Code

素のClaude Code:
  - ファイルを読む
  - ファイルを書く
  - コマンドを実行する
  → これだけ。外部サービスとの連携はゼロ。

今の状態だと、AIはPCの中のファイルしか触れない。外の世界と繋がっていない。

MCPで道具を追加する

Google Sheets MCPを追加 → スプレッドシートを読み書きできるようになる
Web Search MCPを追加   → Web検索ができるようになる
Slack MCPを追加        → Slackに投稿できるようになる
ポイント

App Storeで1つずつアプリを入れるのと同じ。MCPを1個入れたら全機能が使えるわけじゃない。1つずつ。だからリスクも1つずつコントロールできる。

3-4

実例で理解する：60社の会計データ処理

10分

ある税理士がClaude Codeで60社の仕訳を毎晩自動処理している話。

参加者に聞く

「60社の会計データをAIで処理…これ、個人情報的に大丈夫だと思いますか？」

データの流れ

全部のデータがAIに行っているわけじゃない。8割はPC内のキーワード辞書で処理。Anthropicに送るのは判定できなかった2割だけ。しかもAPIで送ったデータはモデルの学習には使われない。

覚えてほしいこと

  • 「AIで処理 ＝ 全データがAI会社に行く」ではない
  • データの流れを設計して、送る範囲を最小限にできる
  • APIとチャットではデータの扱いが違う
3-5

振り返りクイズ

5分
Q1: CLAUDE.mdの正体は？
A: ただのテキストファイル。AIに毎回読ませる「業務マニュアル」。
Q2: MCPを追加するとどうなる？
A: AIの道具箱に新しい道具が1つ追加される。できることが増えるが、アクセス範囲も広がる。
Q3: APIで送信したデータはAIの学習に使われる？
A: 使われない（各社の利用規約に明記）。ブラウザのチャットとは別。

宿題① ルールファイルを書いてみよう（紙でOK）

自分の業務で「AIに伝えたいルール」を5つ書き出してください。

  • 社名は「株式会社○○」で統一
  • 資料のフォントはメイリオ
  • 金額は税込表記
  • 顧客名はダミーに置き換えて使う
  • 敬語は「です/ます」調

宿題② MCPの必要度を考えよう

第1回の宿題で書いた「AIにやらせたい業務」を見返して、以下のどれに当てはまるか考えてください。

  • A. PCの中のファイル操作だけでできそう → MCP不要
  • B. 外部サービスが必要 → MCPが必要
  • C. よく分からない → 第4回で一緒に考えましょう
