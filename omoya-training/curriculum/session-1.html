1-3

チャットAI vs AIエージェント

20分

ここが第1回の核心。

デモ（画面共有）5分

実際にClaude Codeで「簡単な自己紹介ページを作って」と指示。AIがリアルタイムでファイルを作っていく様子を見せる。

RPAとの比較

RPA AIエージェント
たとえると レールの上の電車 自分で道を決めるタクシー
決めたとおりに動く 得意 そもそも決めなくていい
想定外のこと 止まる 自分で考える（たまに間違う）
指示の出し方 フロー図 日本語でOK
ポイント

RPAは安心だけど融通が利かない。AIエージェントは融通が利くけど、どこまで走っていいか決めてあげないと危ない。この「範囲を決める」が今後の講座で学ぶこと。

1-4

AIエージェントツールの種類

10分

全部を覚える必要はない。「こういう道具があるんだ」で十分。

ツール ひとこと たとえると
Claude Code 黒い画面で動く。速い 腕利きの職人
Cursor 画面付き。変更が見える 隣に座る助手
Devin / Antigravity クラウドで全部やる リモートの外注チーム
  • 見た目は違うけど、裏でやっていることは全部同じ（ファイルの読み書き + コマンド実行）
  • この講座で学ぶ考え方は、どのツールにも使える
1-5

振り返りクイズ

10分
Q1: チャットAIとAIエージェントの一番大きな違いは何でしょう？
A: AIが「自分で手を動かす」こと。ファイルを作ったり、コマンドを実行したりする。
Q2: RPAとAIエージェント、どっちが「想定外の対応」に強い？
A: AIエージェント。ただし、想定外の行動をする可能性があるのでリスク管理が大事。
Q3: Claude Code、Cursor、Devinで共通していることは？
A: 裏ではどれも「ファイルの読み書き + コマンドの実行」をしている。
